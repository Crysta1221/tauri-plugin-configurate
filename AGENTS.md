## Commands & permissions

When adding or removing a command, keep these in sync:

- `build.rs` (`COMMANDS`)
- `src/config.rs` (`generate_handler!`)
- `guest-js/configurate.ts` (`invoke("plugin:configurate|...")`)
- `permissions/default.toml`, `permissions/autogenerated/commands/*.toml`, and `permissions/schemas/schema.json` (default set description)

`configurate:allow-unlock` is **not** in `permissions/default.toml`. Apps using `.unlock()` must grant it in capabilities.

## Invariants

- **Storage / keyring order** (`src/commands.rs`): write or delete storage first, then keyring writes/deletes (`execute_create` / `save` / `patch` / `delete`).
- **Validation parity**: mirror rules between `guest-js/configurate.ts` and `src/commands.rs` (`fileName` single component; safe `dirName` / `currentPath`; keyring `id` must not contain `/`).
- **Change events**: keep `configurate://change` aligned between `src/commands.rs` (`ConfigChangeEvent`, `change_target_id`) and `guest-js/configurate.ts` (`ConfigChangeEvent`, `buildChangeTargetId`).
- **Keyring reads**: `keyring_pair` with `KeyringEntryUse::Read` rejects non-empty `value`; writes use `KeyringEntryUse::Write`.
- **Read size limit**: default `DEFAULT_MAX_READ_BYTES` (16 MiB) in `src/config.rs`; override via `Builder::max_read_bytes` or `tauri.conf.json` → `plugins.configurate.maxReadBytes`.
- **Binary `encryptionKey`**: include in IPC only when the operation reads/writes encrypted data (`guest-js/configurate.ts` `_serializeProvider`).

## Releases

User-facing changes go in `CHANGELOG.md` under `## [Unreleased]`.

### Changelog format

Each version block uses Keep a Changelog version headers plus release-note sections:

```markdown
## [Unreleased]

## [0.5.0] - 2026-06-10

# 🚀 0.5.0 Release Notes

One-line summary of the release.

## ⚠️ Breaking Changes
- Use backticks for `types`, `functions()`, and file names.

## ✨ Features
- New behavior and enhancements.

## 🐛 Fixes
- Bug fixes (omit section if none).

## 🔒 Security
- Security hardening (omit section if none).
```

Use `###` only for the emoji sections above — not for Removed/Changed/Added. Wrap identifiers in backticks.

To publish a release:

1. Move `[Unreleased]` content into a new `## [X.Y.Z] - YYYY-MM-DD` block with `# 🚀 X.Y.Z Release Notes`.
2. Bump `version` in `Cargo.toml` and `package.json` to match.
3. Add a new empty `## [Unreleased]` section at the top.
4. Commit, then tag and push: `git tag vX.Y.Z && git push origin vX.Y.Z`.

Pushing a `v*` tag runs `.github/workflows/release.yml`: verify versions → test → build → `cargo publish` → `npm publish` → GitHub Release.

### Publishing credentials

| Target | Method |
| --- | --- |
| crates.io | GitHub Secret `CARGO_REGISTRY_TOKEN` |
| npm | [Trusted Publishing](https://docs.npmjs.com/trusted-publishers/) (OIDC; no `NPM_TOKEN`) |

npm Trusted Publisher on npmjs.com must match this repo (`Crysta1221/tauri-plugin-configurate`) and workflow filename `release.yml`. The workflow sets `id-token: write` for OIDC.

`GITHUB_TOKEN` for the GitHub Release is provided automatically.
